### День 9–10 — Модуль 3: Форма добавления/редактирования товара

**Поля формы:**
- Фото (кнопка выбора файла, заглушка если нет)
- Наименование (текст)
- Категория (выпадающий список из БД)
- Описание (текст)
- Производитель (выпадающий список из БД)
- Поставщик (текст)
- Цена (число, не отрицательное, допустимы копейки)
- Единица измерения (текст)
- Количество на складе (целое, не отрицательное)
- Действующая скидка (число)

**Правила:**
- Добавление: ID не показывается, берётся `MAX(id)+1` из БД
- Редактирование: ID показывается как readonly, все поля заполнены из БД
- Нельзя открыть больше одного окна редактирования
- Фото сохраняется в папку с приложением, размер обрезается до 300×200 px
- При замене фото — старое удаляется из папки
- Нельзя удалить товар если он есть в заказе

**Что сделать:**

- [ ] Валидация полей: цена и количество не отрицательные

### День 11 — Модуль 3: Удаление + обработка ошибок

**Удаление товара:**
- Только администратор
- Перед удалением — предупреждение `QMessageBox.warning`
- Если товар в заказе → `QMessageBox.critical` с объяснением

**Обработка ошибок (везде в приложении):**

- Неверный логин/пароль → сообщение
